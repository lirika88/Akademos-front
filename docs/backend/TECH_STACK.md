# Технологический стек бэкенда

| Категория               | Технология          | Обоснование                                                                 |
|-------------------------|----------------------|-----------------------------------------------------------------------------|
| Среда выполнения        | Node.js              | Высокая производительность, единый язык с фронтендом (JavaScript), огромное сообщество. |
| Веб-фреймворк           | Express              | Минималистичный, гибкий, де-факто стандарт для REST API на Node.js.        |
| База данных             | PostgreSQL           | Надёжная реляционная СУБД, поддержка сложных запросов, целостность данных. |
| ORM (Object-Relational Mapping) | Sequelize    | Удобная работа с моделями, миграции, валидация, поддержка PostgreSQL.      |
| Аутентификация          | JWT (JSON Web Token) | Стандартный механизм для SPA, позволяет хранить токен на клиенте и передавать в заголовках. |
| Хеширование паролей     | bcrypt               | Надёжное хеширование с солью.                                               |
| Валидация данных        | Joi / express-validator | Проверка входящих запросов на соответствие схеме.                           |
| Работа с файлами        | Multer               | Middleware для обработки multipart/form-data (загрузка файлов).              |
| Тестирование            | Jest + Supertest     | Модульные и интеграционные тесты API.                                       |
| Документирование API    | Swagger (OpenAPI)    | Автоматическая генерация документации для удобства фронтенд-разработчика.    |
| Контейнеризация         | Docker               | Унификация окружения разработки и production, упрощение развёртывания.      |
| Оркестрация             | Docker Compose       | Локальный запуск связки бэкенд + БД.                                        |
| CI/CD                   | GitHub Actions       | Автоматизация сборки, тестирования и деплоя.                                |
| Хостинг                 | Render / Heroku / облачный сервер | Простой деплой контейнеризированного приложения.                     |
| Мониторинг              | Winston / Morgan     | Логирование запросов и ошибок.                                              |
