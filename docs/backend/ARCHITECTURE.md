# Архитектура бэкенда

## 1. Общая архитектура проекта (клиент-сервер)

Проект "Akademos" построен по классической клиент-серверной модели:

- **Фронтенд** — статическое одностраничное приложение (SPA) на нативном HTML, CSS, JavaScript. Отвечает за интерфейс и взаимодействие с пользователем.
- **Бэкенд** — монолитное серверное приложение, предоставляющее RESTful API. Обрабатывает запросы, выполняет бизнес-логику, взаимодействует с базой данных и возвращает JSON.

Такое разделение обеспечивает независимость разработки и масштабирования компонентов.

## 2. Выбор архитектуры бэкенда

Для бэкенда выбрана **монолитная архитектура** по следующим причинам:

- Небольшой размер приложения и прогнозируемая нагрузка.
- Простота разработки, развёртывания и отладки.
- Возможность быстро вносить изменения во все связанные модули.
- Все функциональные модули (ученики, материалы, тесты, занятия) тесно связаны и используют общую базу данных, поэтому микросервисная архитектура была бы избыточной.

## 3. Принципы построения API

- API спроектировано в стиле **REST**: ресурсы идентифицируются URL, используются стандартные HTTP-методы (GET, POST, PUT, DELETE).
- Обмен данными осуществляется в формате **JSON**.
- Для аутентификации используется **JWT** (JSON Web Token), который передаётся в заголовке `Authorization: Bearer <token>`.
- Все защищённые эндпоинты проверяют токен и права доступа (роли: репетитор, ученик).
