1. Общая архитектура проекта (клиент-сервер)
Проект "Akademos" представляет собой полноценное веб-приложение, построенное по классической клиент-серверной архитектуре:

Фронтенд (клиентская часть) — статическое одностраничное приложение (SPA), реализованное на нативном HTML, CSS и JavaScript. Отвечает за пользовательский интерфейс и взаимодействие с пользователем. Все запросы к данным отправляются на бэкенд через HTTP.

Бэкенд (серверная часть) — монолитное серверное приложение, предоставляющее RESTful API. Обрабатывает запросы от фронтенда, выполняет бизнес-логику, взаимодействует с базой данных и возвращает ответы в формате JSON.

Такое разделение позволяет разрабатывать и масштабировать компоненты независимо, а также обеспечивает чёткое разделение ответственности.

2. Выбор архитектуры бэкенда
Для бэкенда выбрана монолитная архитектура, что обосновано следующими факторами:

Небольшой размер приложения и прогнозируемая нагрузка.

Простота разработки, развёртывания и отладки.

Возможность быстрого внесения изменений в логику.

Все функциональные модули (ученики, материалы, тесты, занятия) тесно связаны и используют общую базу данных, поэтому выделение микросервисов избыточно.

API строится по принципам REST: ресурсы идентифицируются URL, используются стандартные HTTP-методы (GET, POST, PUT, DELETE), обмен данными осуществляется в формате JSON.
