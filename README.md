# Akademos — Рабочая панель репетитора

**Akademos** — это веб-приложение для организации учебного процесса, созданное специально для репетиторов. Проект состоит из клиентской и серверной частей, которые вместе предоставляют удобный инструмент для ведения учеников, создания заданий, тестов и отслеживания прогресса.

## Общая архитектура

Проект построен по классической клиент-серверной модели:

- **Фронтенд** — одностраничное приложение (SPA) на нативном HTML, CSS и JavaScript. Отвечает за пользовательский интерфейс, взаимодействие с пользователем и отправку запросов к API.
- **Бэкенд** — монолитное серверное приложение на Node.js и Express, предоставляющее RESTful API. Выполняет бизнес-логику, аутентификацию, работу с базой данных и файлами.

Такое разделение позволяет разрабатывать и масштабировать компоненты независимо, а также обеспечивает чёткое разделение ответственности.

## Основные функции

- **Управление учениками** — добавление, редактирование, просмотр профилей и статистики.
- **База материалов** — загрузка и хранение учебных файлов (PDF, Word, видео), организация по папкам.
- **Конструктор тестов** — создание тестов с различными типами вопросов (выбор ответа, открытый вопрос, сопоставление).
- **Выдача домашних заданий** — прикрепление материалов и тестов к ученикам с указанием дедлайна.
- **Проверка и комментарии** — просмотр выполненных работ, выставление оценок, обратная связь.
- **Календарь занятий** — планирование уроков, отметки о проведённых занятиях.

## Быстрый старт

### Запуск фронтенда

Фронтенд — это набор статических файлов. Для локального просмотра достаточно открыть `frontend/index.html` в браузере.

Если вы хотите использовать локальный сервер для разработки (например, с автоматической перезагрузкой), можно воспользоваться любым статическим сервером:

```bash
cd frontend
npx serve .
```

После этого откройте `http://localhost:3000` (или другой порт, который укажет serve).

### Запуск бэкенда

#### Предварительные требования
- Установленные [Docker](https://www.docker.com/) и [Docker Compose](https://docs.docker.com/compose/).
- (Опционально) Node.js 18+ для запуска без Docker.

#### Запуск с Docker

1. Перейдите в папку `backend`:
   ```bash
   cd backend
   ```
2. Скопируйте `.env.example` в `.env` и при необходимости отредактируйте переменные окружения.
3. Запустите контейнеры:
   ```bash
   docker-compose up --build
   ```
4. API будет доступно по адресу `http://localhost:3000/api`.

#### Запуск без Docker (для разработки)

1. Установите зависимости:
   ```bash
   cd backend
   npm install
   ```
2. Настройте подключение к PostgreSQL (например, через локально установленную БД или Docker-контейнер с БД).
3. Создайте базу данных и выполните миграции (Sequelize CLI).
4. Запустите сервер:
   ```bash
   npm run dev
   ```

## CI/CD и деплой

### Фронтенд
- Хостинг: **Netlify**.
- При пуше в ветку `main` происходит автоматический деплой production-версии.
- Pull Request в `develop` создаёт превью-версию для тестирования.

### Бэкенд
- **GitHub Actions** запускает линтеры, тесты и сборку при создании PR.
- Staging-версия автоматически разворачивается на Render/Heroku при слиянии в `develop`.
- Production-версия разворачивается при слиянии в `main` (также через Render/Heroku или другой облачный сервис).

Подробнее — в соответствующих документах: [CI/CD фронтенда](docs/frontend/CI_CD_PIPELINE.md) и [CI/CD бэкенда](docs/backend/CI_CD_PIPELINE.md).
