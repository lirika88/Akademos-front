## Выбор архитектуры проекта

Для данного проекта выбирается классическая монолитная архитектура в рамках клиентской части. Так как приложение строится на нативных технологиях (HTML, CSS, JavaScript), оно будет представлять собой набор статических файлов, связанных общим контекстом (единая точка входа).

**Обоснование:** Проект является многостраничным или одностраничным (SPA) приложением, построенным без использования тяжелых фреймворков. Это обеспечивает простоту развертывания и высокую скорость загрузки. Взаимодействие с сервером построено по принципу клиент-сервер, где фронтенд отправляет запросы к бэкенд-API и динамически обновляет контент на странице.
